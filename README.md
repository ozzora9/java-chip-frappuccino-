# 🌱 StudyPlanner & FocusGarden

**Pomodoro 기반 학습 플래너 + 성장형 정원 시스템**

> 공부한 시간이 실제로 '꽃'으로 자라나는 경험을 제공합니다.

------------------------------------------------------------------------

## 🚀 프로젝트 소개

StudyPlanner는 사용자가 계획을 세우고, 타이머로 공부를 측정하고, 그
데이터를 기반으로 **정원 속 꽃이 성장**하는 구조의 JavaFX 데스크탑
앱입니다.
기록·타이머·정원 시스템이 하나의 데이터 흐름으로 유기적으로 연결되어
있어, 학습 동기 부여를 강화합니다.

------------------------------------------------------------------------

## 🌸 핵심 기능

### 1) 📆 **플래너 페이지**

-   날짜별 학습 계획 등록 
-   일일 기록(메모 / 진행 현황) 저장 
-   SQLite 연동으로 사용자별 히스토리 관리 
-   **기술적 장점:**
    -   캐싱 구조로 페이지 이동 시 재로드 최소화 
    -   JavaFX TableView 커스터마이징 (스타일 + 셀 로직 분리)

------------------------------------------------------------------------

### 2) ⏳ **타이머 페이지**

-   과목 선택 후 집중 시간 측정 
-   타임라인을 통한 **1초 단위 UI 업데이트** 
-   공부 시간에 따라 **꽃 성장 데이터 자동 반영** 
-   **기술적 장점:**
    -   Timeline 기반 1초 루프 최적화 
    -   JSON + DB 혼합 데이터 사용 → FlowerManager가 정적 자원(JSON)을
        제공하고, DatabaseService가 유저별 상태를 제공 
    -   타이머 중 과목 변경 보호 → UI/로직 안정성 확보

------------------------------------------------------------------------

### 3) 🌼 **정원(Garden) 페이지**

-   꽃 배치 / 드래그 / 저장 
-   꽃 3개를 합쳐 상위 꽃으로 업그레이드 
-   사용자별 정원이 DB에 저장되어 앱 재실행에도 유지 
-   **기술적 장점:**
    -   Canvas + ImageView 혼합 렌더링 
    -   '좌표 기반 저장 시스템' 구축 (layoutId, x, y) 
    -   FlowerManager가 이미지 리소스를 중앙 관리하여 유지보수 용이

------------------------------------------------------------------------

### 4) 🌸 **인벤토리 페이지**

-   현재 보유 중인 꽃(seed/flower) 표시 
-   성장 단계(block → seed → sprout → grow → bloom) 이미지 자동 매핑 
-   **기술적 장점:**
    -   JSON 기반 정적 정보 + DB 기반 동적 보유량을 분리 
    -   카드 UI 공통 CSS 적용으로 일관성 유지 
    -   GridPane 자동 정렬 레이아웃 구성

------------------------------------------------------------------------

## 🗃 데이터 구조

### 🔧 사용 기술

-   **Java 23**
-   **JavaFX 21**
-   **SQLite**
-   **Gradle (선택 이유: 빌드 속도 + 설정 간결함 + 최신 Java와의 궁합)**

### 🔌 외부 라이브러리

-   Gson (JSON 파싱)
-   SQLite JDBC 드라이버
-   FXControls / JFoenix 일부 UI 적용 가능 (선택)

------------------------------------------------------------------------

### 📁 JSON + DB 혼용 이유

-   **JSON**: 꽃 정보(정적, 변하지 않음) → 이미지 경로, 꽃말, 단계
    이미지 
-   **DB**: 사용자별 동적 정보 → 보유 수량, 잠금 여부, 배치 좌표

→ 정적/동적 데이터 분리를 통해 **리소스 관리 최적화 + 유지보수성 증가**

------------------------------------------------------------------------

## 🌱 성장 시스템 개요

-   타이머 기록 → DailyRecord 저장 
-   누적 시간에 따라 **꽃 성장률 계산** 
-   성장 단계별 이미지 UI 자동 변경 
-   임계치 도달 시 seed→sprout→grow→bloom 순으로 진화 
-   3개 bloom 조합 = 상위 희귀 꽃 생성

------------------------------------------------------------------------

  페이지     주요 기능         기술적 장점
  ---------- ----------------- -----------------------------
